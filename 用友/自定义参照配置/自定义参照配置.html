    # 3.重写Refer查询器.

    上面说到的datasourcetype,这个字段,会决定程序到哪里去找你的refer查询器.下面详细看看.

    源码里RefService,这个getGridData()方法,里面有段代码,是说去找默认的方法还是自定义方法的.

    //这是摘的源码
if (entity.datasourceType == null || entity.datasourceType.isEmpty() || entity.datasourceType.equalsIgnoreCase(MddConstants.TYPE_ENTITY_DATASOURCE_META)) {
            String datasourceType = MddConstants.TYPE_ENTITY_DATASOURCE_META;
            // 获取指定名称的IRefDataHandler实现类，name命名规则为 entity.datasourceType + "RefDataHandler"
            IExtRefDataHandler refDataHanlder = AppContext.getBean(datasourceType + "ExtRefDataHandler", IExtRefDataHandler.class);

    截图看看:

    看,如果你没有指定,或者指定的meta,他都会走默认的,MetaExtRefDataHandler,如果你重写了,那就是会走你自己的datasourceType+ExtRefDataHandler,自定义参照查询器.

    需要注意的是,如果自己写查询器,最后要返回一个Pager,这里有分页信息,需要自己处理下.

    Pager pager = new Pager();
pager.setPageIndex(1);
pager.setPageSize(20);
pager.setRecordCount(wbsTemplateDtoList.size());
pager.setRecordList(wbsTemplateDtoList);
pager.setPageCount(1);
return pager;

    可以参照wbsTplRefExtRefDataHandler这个类,在计划中台.

    # 4.自定义Refer查询器注意的点.

    1. 除了自己的条件需要处理以外.

      自己的参数都在实现接口的map参数的condition中,map.get("condition");

    2. 需要注意模糊查询的处理.

      模糊查询就是重新调用接口,但是参数直接在map中.

    3. 除了模糊查询,还有精准查询,精准查询的条件也在map.get("condition");中,但是取时候要注意,他是自己封装了一个conditions
