# 一.ThreadClearFilter

注册过滤器的类.这个类返回了一个过滤器bean,如果有其他的过滤器可以配置多少@Bean,当前是注册个线程信息刷新过滤器.ThreadClearFilter,让我们看看这个过滤器内部.

@Override
public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) throws IOException, ServletException {
    try {
        chain.doFilter(request, response);
    } finally {
        RuntimeContext.resetCurrentThread();
        BizContext.resetCurrentThread();
        ThreadLockContext.releaseCurrThreadLock();
    }
}

从上面的代码可以看出其实在开始什么都没做,而是在结束后做了些事,

  1. 清除运行时上下文

  2. 清除业务上下文

  3. 清除此次所有的锁

    当然前提是你的锁必须是通过ThreadLockContext上的才可以,它内部就是个ThreadLocal<Stack<Object>> threadStatckInfo

# 二.ServiceContextInterceptor

初始化运行环境的,就是上面的过滤器最后第一步清理的那个

public static void initByAppContext(){
        TraceInfo traceInfo = getTraceInfo();
        RuntimeContext.initCurrentThread(getToken(),getUserId(),getTenantId(),traceInfo);
    }

最核心的就是设置一些初始信息

 RuntimeContext类
public static void initCurrentThread(String token ,String userId, String tenantId, TraceInfo traceInfo){
        resetCurrentThread();
        setToken(token);
        setLoginUserId(userId);
        setTenantId(tenantId);
        setTraceInfo(traceInfo);
    }

# 三.PMAuthInterceptor

项目云功能权限校验,

  1. 校验SERVICE_CODE对应的应用编码是否存在,
  2. 是否有登录信息,没有抛异常
  3. 如果是主管理员类,直接通过所有权限
  4. 否则查看他的权限,是否越权,越权报错

# 四.PMReSubmitInterceptor

这是一个检查重复提交的拦截器

  1. 如果request里或者参数里都没有,那就认为他不需要这个操作,直接返回了
  2. 如果有,看看缓存里有没有AppContext.cache().get(resubmitCheckKey),如果有了,那就是重复提交了,抛异常
  3. 否则尝试加锁,然后向缓存里加入这个key,value为当前时间,毫秒,
  4. 最后释放锁
